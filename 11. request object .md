# What this teaches you

The request object is not just body data.

It is a **live container** holding:

* transport details
* auth state
* user identity
* headers
* parsed payload
* server metadata
* middleware-added context

---

# Most used parts in real apps

```python id="x8q4rt"
request.user
request.data
request.headers
request.META
request.session
request.COOKIES
request.FILES
```

---

# In middleware, services, views

You pull exactly what you need.

Example:

```python id="k2z7mw"
ip = request.META.get("HTTP_X_FORWARDED_FOR")
user = request.user
payload = request.data
```

# Final mental model

```text id="v6n3pd"
HTTP request from client
        ↓
Django parses it
        ↓
Builds request object
        ↓
Middleware can modify it
        ↓
Views consume it
```

---

That object is the heartbeat of every Django request cycle.
