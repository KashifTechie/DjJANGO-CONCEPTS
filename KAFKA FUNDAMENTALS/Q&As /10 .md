# Kafka Producer

```python id="u1c4jx"
from confluent_kafka import Producer
```

Producer sends messages/events to Kafka topics.

Example:

* email jobs
* webhook events
* logs
* analytics
* notifications

---

# Producer Lifecycle

```text id="g6p5kt"
Create Producer
    ↓
produce()
    ↓
Kafka internal queue
    ↓
Broker receives
    ↓
delivery callback
    ↓
flush()
```

---

# Producer Methods

---

## 1. Producer()

Creates producer instance.

```python id="a6j9hx"
producer = Producer(config)
```

Example:

```python id="w3ixl4"
producer = Producer({
    "bootstrap.servers": "localhost:9092"
})
```

---

## 2. produce()

MOST IMPORTANT METHOD.

Sends message asynchronously.

```python id="3ubg7r"
producer.produce(
    topic,
    value,
    key=None,
    partition=None,
    callback=None,
    headers=None
)
```

Example:

```python id="cxq0vb"
producer.produce(
    "emails",
    key="user123",
    value='{"email":"a@test.com"}'
)
```

### What It Does

* serializes message
* puts into internal producer buffer
* Kafka broker receives later
* async/non-blocking

---

## 3. poll()

Processes delivery callbacks/events.

```python id="v4g8ow"
producer.poll(0)
```

### What It Does

* triggers delivery callbacks
* processes broker acknowledgements
* frees internal queue memory

Usually called after produce.

Example:

```python id="36jlwm"
producer.produce(...)
producer.poll(0)
```

---

## 4. flush()

Waits until all messages are delivered.

```python id="8t5lpk"
producer.flush()
```

### What It Does

* blocks until queue empty
* ensures all messages sent
* required before shutdown

Example:

```python id="t5mwz0"
producer.flush(timeout=10)
```

---

## 5. purge()

Drops queued/in-flight messages.

```python id="j2r4y4"
producer.purge()
```

### What It Does

* clears producer queue
* removes unsent messages

Used during crashes/shutdowns.

---

## 6. len(producer)

Returns queued message count.

```python id="rqm77r"
len(producer)
```

### What It Does

Shows:

* internal backlog
* queue pressure

Useful for monitoring.

---

## 7. list_topics()

Gets cluster metadata.

```python id="tx3nfx"
producer.list_topics()
```

### What It Does

Returns:

* topics
* partitions
* brokers

---

## 8. Delivery Callback

```python id="cd54p5"
def delivery_report(err, msg):
    if err:
        print("Failed")
    else:
        print("Delivered")
```

Used in:

```python id="1brx6l"
producer.produce(
    "emails",
    value="hello",
    callback=delivery_report
)
```

### What It Does

Executes when:

* message delivered
* delivery failed

---

# Producer Concepts

---

## Key

```python id="ny8r0w"
key="user123"
```

### What It Does

Determines partition.

Same key:

* same partition
* preserves order

---

## Partition

Kafka topic split into partitions.

Example:

```text id="8x02vb"
emails_topic
    partition0
    partition1
    partition2
```

Producer sends messages into partitions.

---

## Batching

Kafka groups messages together.

Controlled by:

* linger.ms
* batch.size

### What It Does

* improves throughput
* reduces network calls

---

## Compression

```python id="spq1hb"
compression.type=gzip
```

### What It Does

Compresses batches before sending.

---

## Idempotence

```python id="mjlwm7"
enable.idempotence=true
```

### What It Does

Prevents duplicate messages.

Important for production.

---

# Consumer Methods

---

## 1. Consumer()

Creates consumer instance.

```python id="ttxjlwm"
consumer = Consumer(config)
```

Example:

```python id="92lx1v"
consumer = Consumer({
    "bootstrap.servers": "localhost:9092",
    "group.id": "email-workers"
})
```

---

## 2. subscribe()

Subscribes to topics.

```python id="wvbqj1"
consumer.subscribe(["emails"])
```

### What It Does

Kafka automatically:

* assigns partitions
* handles rebalancing

---

## 3. poll()

MOST IMPORTANT METHOD.

Fetches messages.

```python id="k36f9t"
msg = consumer.poll(1.0)
```

### What It Does

* fetches message
* sends heartbeats
* maintains group membership

---

## 4. consume()

Fetches multiple messages.

```python id="0c6vns"
messages = consumer.consume(num_messages=100)
```

### What It Does

Batch fetch for:

* higher throughput
* bulk processing

---

## 5. commit()

Marks offsets processed.

```python id="mwv9ww"
consumer.commit()
```

OR:

```python id="j4h2so"
consumer.commit(message=msg)
```

### What It Does

Tells Kafka:

```text id="nghkl6"
this message processed successfully
```

---

## 6. close()

Gracefully shuts down consumer.

```python id="8zk5pz"
consumer.close()
```

### What It Does

* leaves consumer group
* commits offsets
* releases partitions

VERY IMPORTANT.

---

## 7. pause()

Temporarily stops fetching.

```python id="xkzh6z"
consumer.pause(consumer.assignment())
```

### What It Does

* stops fetching new messages
* keeps consumer alive

Used for:

* backpressure
* rate limiting

---

## 8. resume()

Resumes fetching.

```python id="p8ch8m"
consumer.resume(consumer.assignment())
```

### What It Does

Continues consumption after pause.

---

## 9. assignment()

Gets assigned partitions.

```python id="2ryhji"
consumer.assignment()
```

Example output:

```python id="gjcaz5"
[
    TopicPartition("emails", 0),
    TopicPartition("emails", 1),
]
```

---

## 10. seek()

Moves offset manually.

```python id="74lj2h"
consumer.seek(
    TopicPartition("emails", 0, 500)
)
```

### What It Does

Jump to:

* old messages
* replay messages
* retry processing

---

## 11. position()

Gets current offsets.

```python id="2chjlwm"
consumer.position(partitions)
```

### What It Does

Shows:

* current read position

---

## 12. committed()

Gets committed offsets.

```python id="gk1t0p"
consumer.committed(partitions)
```

### What It Does

Shows saved offsets in Kafka.

---

## 13. offsets_for_times()

Get offsets by timestamp.

```python id="3ucskm"
consumer.offsets_for_times(...)
```

### What It Does

Read messages from:

* specific time/date

Useful for replay systems.

---

## 14. list_topics()

Gets cluster metadata.

```python id="xjlwm0"
consumer.list_topics()
```

---

# Important Kafka Concepts

---

## Consumer Group

Multiple consumers sharing work.

Example:

```text id="jlwm44"
6 partitions
3 consumers
```

Result:

```text id="jlwm55"
2 partitions each
```

---

## Rebalancing

Occurs when:

* consumer joins
* consumer leaves
* crash happens

Kafka redistributes partitions.

---

## Offset

Kafka message position.

Example:

```text id="jlwm66"
Partition 0
Offset 100
```

Means:

* consumer processed 100 messages

---

## Auto Commit

```python id="jlwm77"
enable.auto.commit=True
```

Kafka commits automatically.

Easy but risky.

---

## Manual Commit

```python id="jlwm88"
enable.auto.commit=False
```

Commit after successful processing.

Production standard.

# Important Producer Configs

| Config             | Purpose            |
| ------------------ | ------------------ |
| bootstrap.servers  | Kafka brokers      |
| acks               | Delivery guarantee |
| retries            | Retry failures     |
| linger.ms          | Batch wait         |
| batch.size         | Batch size         |
| compression.type   | Compression        |
| enable.idempotence | Prevent duplicates |

---

# Important Consumer Configs

| Config               | Purpose            |
| -------------------- | ------------------ |
| group.id             | Consumer group     |
| auto.offset.reset    | Start position     |
| enable.auto.commit   | Offset handling    |
| max.poll.interval.ms | Processing timeout |
| session.timeout.ms   | Heartbeat timeout  |
