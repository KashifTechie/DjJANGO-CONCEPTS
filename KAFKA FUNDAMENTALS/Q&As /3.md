### ❓ Question

In **Apache Kafka**, when a producer calls `send()` multiple times to send messages to a topic, how are **offsets assigned** to those messages inside a partition?

---

### ✅ Answer

In Kafka, every message written to a **partition** is assigned a **unique sequential offset** that represents its position in the partition log.

When a producer calls `send()`:

1. The producer sends the message to the Kafka broker.
2. The broker determines the **target partition**.
3. When the broker **appends the message to the partition log**, it assigns the **next available offset**.
4. Each new message in that partition receives an offset that is **one greater than the previous message’s offset**.

---

### 📊 Example

Producer code:

```python
producer.send("orders", value="order1")
producer.send("orders", value="order2")
producer.send("orders", value="order3")
```

If all messages go to **partition 0**, the log will look like:

```
Partition 0

Offset   Message
0        order1
1        order2
2        order3
```

Here:

* First `send()` → offset **0**
* Second `send()` → offset **1**
* Third `send()` → offset **2**

Each message gets its **own offset**.

---

### ⚠️ Important Detail

Offsets are **not global for the topic** — they are **specific to each partition**.

Example with two partitions:

```
Partition 0
Offset   Message
0        order1
1        order3

Partition 1
Offset   Message
0        order2
```

Each partition maintains its **own independent offset sequence**.

---

### 🧠 Key Takeaway

* **1 message appended to a partition = 1 offset**
* Offsets increase sequentially
* Offsets exist **per partition**, not per topic
* Offsets are assigned **when the broker writes the message to the log**
