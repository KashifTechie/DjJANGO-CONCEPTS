# Arrow Functions

---

# 🔹 1. What is an Arrow Function?

A **shorter way to write functions** introduced in ES6.

```javascript
// Regular Function
function add(a, b) {
    return a + b;
}

// Arrow Function — same thing, shorter syntax
const add = (a, b) => {
    return a + b;
};
```

---

# 🔹 2. Syntax Variations

Arrow functions have **different shorthand rules** depending on the situation.

---

### ➤ Full Syntax — Multiple Parameters, Multiple Lines

```javascript
const add = (a, b) => {
    let result = a + b;
    return result;
};

console.log(add(3, 5)); // 8
```

---

### ➤ Implicit Return — Single Expression

When the function body is **one line**, you can remove `{}` and `return`.

```javascript
// Full
const add = (a, b) => {
    return a + b;
};

// Shorthand — same result
const add = (a, b) => a + b;

console.log(add(3, 5)); // 8
```

---

### ➤ Single Parameter — No Parentheses Needed

```javascript
// With parentheses
const square = (n) => n * n;

// Without parentheses — both are valid
const square = n => n * n;

console.log(square(4)); // 16
```

---

### ➤ No Parameters — Empty Parentheses Required

```javascript
const greet = () => "Hello, World!";

console.log(greet()); // Hello, World!
```

---

### ➤ Returning an Object — Wrap in Parentheses

```javascript
// ❌ BAD — JS thinks {} is a code block, not an object
const getUser = () => { name: "Kashif" };

// ✅ GOOD — wrap object in ()
const getUser = () => ({ name: "Kashif", age: 22 });

console.log(getUser()); // { name: "Kashif", age: 22 }
```

# 🔹 3. All 4 Scenarios

---

### ➤ Scenario 1 — With Parameters and Return

```javascript
const multiply = (a, b) => a * b;

console.log(multiply(4, 5)); // 20
```

---

### ➤ Scenario 2 — With Parameters and No Return

```javascript
const printName = (name) => {
    console.log("Hello, " + name);
};

printName("Kashif"); // Hello, Kashif
```

---

### ➤ Scenario 3 — Without Parameters and Return

```javascript
const getCurrentYear = () => new Date().getFullYear();

console.log(getCurrentYear()); // 2026
```

---

### ➤ Scenario 4 — Without Parameters and No Return

```javascript
const showDivider = () => {
    console.log("--------------------");
};

showDivider(); // --------------------
```

# 🔹 4. Arrow Function vs Regular Function

---

### ➤ Syntax Difference

```javascript
// Regular
function add(a, b) {
    return a + b;
}

// Arrow
const add = (a, b) => a + b;
```

---

### ➤ `this` Keyword Difference — Most Important Difference

Regular functions have their **own** `this`.
Arrow functions **inherit** `this` from the surrounding scope.

```javascript
// Regular Function — this is the object
const person = {
    name: "Kashif",
    greet: function() {
        console.log("Hello, " + this.name);
    }
};

person.greet(); // Hello, Kashif
```

```javascript
// Arrow Function — this is NOT the object
const person = {
    name: "Kashif",
    greet: () => {
        console.log("Hello, " + this.name);
    }
};

person.greet(); // Hello, undefined ❌
```

---

### ➤ Where Arrow `this` is Useful — Inside Callbacks

```javascript
function Timer() {
    this.seconds = 0;

    // ❌ Regular function — this is lost inside setInterval
    setInterval(function() {
        this.seconds++;
        console.log(this.seconds); // NaN — wrong this
    }, 1000);
}

function Timer() {
    this.seconds = 0;

    // ✅ Arrow function — this is inherited from Timer
    setInterval(() => {
        this.seconds++;
        console.log(this.seconds); // 1, 2, 3... — correct
    }, 1000);
}
```

---

### ➤ No `arguments` Object

Regular functions have a built-in `arguments` object.
Arrow functions do **not**.

```javascript
// Regular — arguments works
function showArgs() {
    console.log(arguments);
}

showArgs(1, 2, 3); // [1, 2, 3]


// Arrow — arguments does NOT work
const showArgs = () => {
    console.log(arguments); // ❌ ReferenceError
};

// ✅ Use rest parameters instead
const showArgs = (...args) => {
    console.log(args); // [1, 2, 3]
};

showArgs(1, 2, 3);
```

# 🔹 5. Arrow Functions with Array Methods

Arrow functions are **most commonly used** as callbacks inside array methods.

---

### ➤ `map()`

```javascript
let numbers = [1, 2, 3, 4];

let doubled = numbers.map(n => n * 2);

console.log(doubled); // [2, 4, 6, 8]
```

---

### ➤ `filter()`

```javascript
let numbers = [1, 2, 3, 4, 5, 6];

let evens = numbers.filter(n => n % 2 === 0);

console.log(evens); // [2, 4, 6]
```

---

### ➤ `reduce()`

```javascript
let numbers = [1, 2, 3, 4];

let sum = numbers.reduce((acc, curr) => acc + curr, 0);

console.log(sum); // 10
```

---

### ➤ `find()`

```javascript
let users = [
    { id: 1, name: "Ali" },
    { id: 2, name: "Kashif" }
];

let user = users.find(u => u.id === 2);

console.log(user); // { id: 2, name: "Kashif" }
```

---

# 🔹 6. When to Use and When NOT to Use

| Situation | Use Arrow? |
|-----------|------------|
| Short, simple functions | ✅ Yes |
| Callbacks inside map, filter, reduce | ✅ Yes |
| When `this` should be inherited | ✅ Yes |
| Object methods that use `this` | ❌ No |
| Functions that need `arguments` object | ❌ No |
| Constructor functions (`new` keyword) | ❌ No |

---

# 🔥 Final Summary Table

| Feature | Regular Function | Arrow Function |
|---------|-----------------|----------------|
| Syntax | `function fn() {}` | `const fn = () => {}` |
| `this` | Own `this` | Inherits from parent |
| `arguments` object | ✅ Yes | ❌ No |
| Can use as constructor | ✅ Yes | ❌ No |
| Implicit return | ❌ No | ✅ Yes (single line) |
| Best used for | Object methods | Callbacks, short functions |

---

# 💡 Key Takeaway

- Arrow functions are **shorter** but not a full replacement for regular functions
- Use arrow functions for **callbacks and short expressions**
- Use regular functions when you need **`this`** to refer to the object
- Single expression → **drop `{}` and `return`** for cleaner code
- Returning an object → always **wrap in `()`**
