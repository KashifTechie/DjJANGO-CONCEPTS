# Control Flow

---

# 🔹 1. if / else

Used to **execute code based on a condition** — if the condition is true, one block runs; otherwise another block runs.

---

### ➤ Basic `if`

```javascript
let age = 20;

if (age >= 18) {
    console.log("Adult");
}
// Adult
```

---

### ➤ `if` / `else`

```javascript
let age = 15;

if (age >= 18) {
    console.log("Adult");
} else {
    console.log("Minor");
}
// Minor
```

---

### ➤ `if` / `else if` / `else`

```javascript
let score = 75;

if (score >= 90) {
    console.log("A grade");
} else if (score >= 75) {
    console.log("B grade");
} else if (score >= 60) {
    console.log("C grade");
} else {
    console.log("Fail");
}
// B grade
```

---

### ➤ Nested `if`

```javascript
let age = 20;
let hasID = true;

if (age >= 18) {
    if (hasID) {
        console.log("Entry allowed");
    } else {
        console.log("No ID, entry denied");
    }
} else {
    console.log("Too young");
}
// Entry allowed
```

---

### ➤ Shorthand — Ternary Operator

Use when condition is **simple and has only two outcomes**.

```javascript
// Syntax: condition ? valueIfTrue : valueIfFalse

let age = 20;

let result = age >= 18 ? "Adult" : "Minor";

console.log(result); // Adult
```

---

### ➤ Chained Ternary

```javascript
let score = 75;

let grade = score >= 90 ? "A"
           : score >= 75 ? "B"
           : score >= 60 ? "C"
           : "Fail";

console.log(grade); // B
```

> ⚠️ Avoid deep chaining — use `if/else if` for better readability.

---

### ➤ Truthy and Falsy Values

JavaScript automatically treats some values as `true` or `false` in conditions.

**Falsy values** (treated as `false`):

```javascript
false
0
""        // empty string
null
undefined
NaN
```

**Truthy values** (everything else):

```javascript
true
1
"hello"
[]        // even empty array
{}        // even empty object
```

**Example:**

```javascript
let name = "";

if (name) {
    console.log("Name exists");
} else {
    console.log("Name is empty");
}
// Name is empty
```

---

### ➤ Logical Operators in Conditions

#### `&&` → AND (both must be true)
```javascript
let age = 20;
let hasID = true;

if (age >= 18 && hasID) {
    console.log("Entry allowed");
}
// Entry allowed
```

#### `||` → OR (at least one must be true)
```javascript
let isAdmin = false;
let isModerator = true;

if (isAdmin || isModerator) {
    console.log("Access granted");
}
// Access granted
```

#### `!` → NOT (reverses the condition)
```javascript
let isLoggedIn = false;

if (!isLoggedIn) {
    console.log("Please log in");
}
// Please log in
```

# 🔹 2. switch

Used when you need to **compare one value against many possible cases** — cleaner than multiple `else if` for this purpose.

---

### ➤ Basic switch

```javascript
let day = "Monday";

switch (day) {
    case "Monday":
        console.log("Start of work week");
        break;
    case "Friday":
        console.log("End of work week");
        break;
    case "Saturday":
    case "Sunday":
        console.log("Weekend");
        break;
    default:
        console.log("Midweek");
}
// Start of work week
```

---

### ➤ How it Works Step by Step

```
1. switch(value) → takes the value to compare
2. case "x":    → checks if value === "x"
3. break        → exits the switch (IMPORTANT)
4. default:     → runs if no case matches (like else)
```

---

### ➤ What Happens Without `break` — Fall-through

```javascript
let num = 1;

switch (num) {
    case 1:
        console.log("One");
        // ❌ no break → falls through
    case 2:
        console.log("Two");
        break;
    case 3:
        console.log("Three");
        break;
}
// One
// Two  ← also runs because of missing break
```

> ⚠️ Always add `break` unless fall-through is intentional.

---

### ➤ Grouping Cases (Intentional Fall-through)

```javascript
let fruit = "banana";

switch (fruit) {
    case "apple":
    case "banana":
    case "mango":
        console.log("This is a fruit");
        break;
    case "carrot":
    case "broccoli":
        console.log("This is a vegetable");
        break;
    default:
        console.log("Unknown item");
}
// This is a fruit
```

---

### ➤ switch with Numbers

```javascript
let score = 2;

switch (score) {
    case 1:
        console.log("Poor");
        break;
    case 2:
        console.log("Average");
        break;
    case 3:
        console.log("Good");
        break;
    default:
        console.log("Invalid score");
}
// Average
```

---

### ➤ switch with return (inside a function)

When inside a function, you can use `return` instead of `break`.

```javascript
function getDayType(day) {
    switch (day) {
        case "Saturday":
        case "Sunday":
            return "Weekend";
        default:
            return "Weekday";
    }
}

console.log(getDayType("Saturday")); // Weekend
console.log(getDayType("Monday"));   // Weekday
```

# 🔹 3. if / else vs switch — When to Use What

| Situation | Use |
|-----------|-----|
| Checking ranges (`> 10`, `< 50`) | `if / else` |
| Checking complex conditions (`&&`, `\|\|`) | `if / else` |
| Comparing one value to many exact values | `switch` |
| Cleaner code for 4+ fixed cases | `switch` |

---

### ➤ Example — Same logic, both ways

```javascript
// Using if / else
let role = "admin";

if (role === "admin") {
    console.log("Full access");
} else if (role === "editor") {
    console.log("Edit access");
} else if (role === "viewer") {
    console.log("Read only");
} else {
    console.log("No access");
}

// Using switch → cleaner for this case
switch (role) {
    case "admin":
        console.log("Full access");
        break;
    case "editor":
        console.log("Edit access");
        break;
    case "viewer":
        console.log("Read only");
        break;
    default:
        console.log("No access");
}
```

---

# 🔥 Final Summary Table

| Feature | if / else | switch |
|---------|-----------|--------|
| Checks ranges | ✅ | ❌ |
| Checks exact values | ✅ | ✅ |
| Multiple conditions (`&&`, `\|\|`) | ✅ | ❌ |
| Cleaner for many fixed cases | ❌ | ✅ |
| Fall-through behavior | ❌ | ✅ (with missing `break`) |
| Default / fallback | `else` | `default` |

---

# 💡 Key Takeaway

- Use **`if / else`** for ranges, complex conditions, or simple two-path logic
- Use **`switch`** when comparing one value against many fixed options
- Always add **`break`** in switch cases unless fall-through is intentional
- Use **ternary** only for short, simple two-outcome conditions
